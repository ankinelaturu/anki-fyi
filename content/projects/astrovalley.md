# AstroValley

A browser-native AI sandbox disguised as an astrology application.

## What It Is

AstroValley is a local-AI experimentation platform built around a deliberately unusual domain: astrology.

At first glance it appears to be an astrology product, but the real purpose is much broader. AstroValley serves as a sandbox for exploring local LLM orchestration, browser-native AI experiences, procedural world generation, semantic interactions, and rich visual interfaces that run close to the user.

The application uses astrology because it provides a highly structured data model containing planets, houses, signs, timelines, relationships, and symbolic meaning. That structure makes it an excellent environment for testing AI-driven experiences without starting from a blank slate.

The long-term vision is a personalized interactive world where data, visualization, AI reasoning, and procedural generation work together inside the browser.

## Why I Built It

I wanted a project that could combine several of the topics I had been exploring independently:

- Local LLM execution
- Browser runtimes
- AI orchestration
- Interactive 3D environments
- Deterministic world generation
- Rich visual storytelling

Astrology happened to provide a surprisingly useful framework because it contains structured entities, relationships, timelines, and interpretations that can be transformed into visual and conversational experiences.

Rather than building another chat application, I wanted to explore what AI feels like when it becomes part of a living environment.

## Technical Highlights

- Browser-native architecture.
- React + Next.js application foundation.
- React Three Fiber and Three.js rendering.
- WebGPU and WebLLM experimentation.
- Local Gemma model direction.
- astronomy-engine integration.
- Deterministic world generation.
- Persistent browser-side state.
- Seed-based procedural content generation.
- AI-assisted interaction layers built on structured domain data.

## Key Features

- Personalized valley generation.
- Deterministic worlds derived from user profile data.
- Procedural placement of houses and structures.
- Interactive planetary systems.
- Local-AI experimentation environment.
- Browser-side persistence.
- Rich 3D exploration experience.
- Visual representation of structured astrological data.

## Technical Challenges

One challenge is balancing playfulness with technical rigor. The experience should feel imaginative while still being built on deterministic systems that can be reproduced consistently.

Another challenge is local AI execution. Browser-side models introduce concerns around download size, GPU capabilities, memory limits, caching, and progressive loading.

Procedural generation creates a third challenge: generating worlds that feel unique while remaining deterministic and explainable.

## Why It Matters

AstroValley represents many of the themes that appear throughout my recent work.

It combines local AI, browser runtimes, procedural interfaces, structured data systems, and interactive visualization into a single platform.

The interesting part is not astrology itself. The interesting part is using a structured symbolic domain to explore what browser-native AI experiences might look like in the future.

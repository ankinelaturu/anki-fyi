# Oracle Mobile Application Framework

## What It Is

Oracle Mobile Application Framework, or MAF, was a cross-platform mobile framework for building enterprise mobile applications across iOS, Android, and Windows platforms.

The work involved framework architecture, runtime components, Java integration, native mobile layers, and cross-platform reliability.

## Time Period

**Dec 2013 - Jul 2018**

## Role

**Principal Member of Technical Staff, Oracle**

**Architect - Oracle Mobile Application Framework (MAF)**

## What I Did

I contributed to the architecture and development of a cross-platform mobile framework supporting iOS, Android, and Windows platforms.

I led migration efforts to Java 8 and modern runtime integrations, and designed runtime components across Java, C/C++, and native mobile layers.

The work focused on performance optimization, runtime integration, cross-platform reliability, and enterprise mobile application support.

## Technical Highlights

- Contributed to architecture of a cross-platform mobile framework.
- Supported iOS, Android, and Windows platforms.
- Led migration to Java 8.
- Designed runtime components across Java, C/C++, and native mobile layers.
- Focused on performance optimization and runtime reliability.
- Worked across embedded runtime and mobile application boundaries.
- Delivered technical sessions at JavaOne and Oracle OpenWorld.

## Key Features / Outcomes

- Helped enable enterprise mobile applications across multiple platforms.
- Modernized runtime support through Java 8 migration.
- Improved cross-platform reliability and runtime integration.
- Supported native mobile interoperability.
- Contributed to a framework used by enterprise developers building mobile applications.

## Technical Challenges

Cross-platform mobile frameworks must hide platform differences without eliminating the unique constraints of each operating system.

The framework needed to support enterprise-grade applications while dealing with runtime integration, native mobile APIs, Java execution, performance constraints, and differences across iOS, Android, and Windows.

## Why It Matters

This experience deepened my understanding of runtime systems, platform abstraction, and cross-platform developer tooling.

It also connects to my later browser-native and local-first AI work. In both cases, the challenge is similar: make complex runtime capabilities available through a developer-friendly platform.
